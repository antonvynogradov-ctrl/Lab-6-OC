# Словник термінів

**Архівування** — процес об'єднання декількох файлів або каталогів в один архівний файл.

**Стиснення** — процес зменшення розміру файлу за допомогою спеціальних алгоритмів.

**Командна оболонка (Shell)** — програма, що дозволяє користувачу взаємодіяти з операційною системою за допомогою текстових команд.

**Потоки (Streams)** — канали передачі даних між програмами.

**Перенаправлення (Redirection)** — зміна напрямку стандартних потоків введення або виведення.

**Pipe (канал)** — механізм передачі результату виконання однієї команди як вхідних даних для іншої.

### 2. Команди з лабораторних робіт

| Назва команди | Її призначення та функціональність |
| :--- | :--- |
| `mkdir mybackups` | Створення нової директорії **mybackups** у домашньому каталозі користувача. |
| `tar -cvf mybackups/udev.tar /etc/udev` | Команда **tar** використовується для об'єднання кількох файлів в один архів. В даному випадку вміст директорії **/etc/udev** буде збережено в архів **udev.tar** у директорії **mybackups**. Параметр **-c** (create) — створити архів, **-v** (verbose) — детальний вивід, **-f** (file) — вказівка імені файлу. |
| `tar -xvf archive.tar` | Витягує (розпаковує) вміст архіву `archive.tar` у поточну директорію. Параметр **-x** (extract) означає витягнення. |
| `tar -tvf archive.tar` | Виводить список файлів, що містяться в архіві `archive.tar`, без їх розпакування. Параметр **-t** (list) означає перегляд вмісту. |
| `gzip file1` | Стискає файл `file1` за допомогою алгоритму gzip. Створює файл `file1.gz` та зазвичай видаляє оригінальний нестиснений файл. |
| `gunzip file1.gz` | Розпаковує файл, стиснений за допомогою gzip, відновлюючи оригінальний `file1`. |
| `bzip2 file1` | Стискає файл за допомогою алгоритму bzip2 (створює `file1.bz2`). Забезпечує щільніше стиснення, ніж gzip, але працює повільніше. |
| `bunzip2 file1.bz2` | Розпаковує файл, стиснений за допомогою bzip2. |
| `xz file1` | Стискає файл алгоритмом xz (створює `file1.xz`). Забезпечує найвищий рівень стиснення серед стандартних утиліт Linux. |
| `unxz file1.xz` | Розпаковує файл, стиснений за допомогою xz. |
| `tar -czvf arch.tar.gz /dir` | Створює архів директорії `/dir` та одразу стискає його за допомогою **gzip** (параметр **-z**). |
| `tar -cjvf arch.tar.bz2 /dir` | Створює архів директорії `/dir` та одразу стискає його за допомогою **bzip2** (параметр **-j**). |
| `zip archive.zip file1 file2` | Створює стиснений архів формату ZIP, додаючи до нього `file1` та `file2`. |
| `unzip archive.zip` | Розпаковує архів формату ZIP у поточну директорію. |
| `cat file.txt` | Виводить весь вміст текстового файлу `file.txt` на екран термінала (стандартне виведення). |
| `less file.txt` | Дозволяє посторінково переглядати вміст великих текстових файлів з можливістю прокрутки вгору та вниз. |
| `head file.txt` | Виводить перші 10 рядків файлу (за замовчуванням). |
| `head -n 5 file.txt` | Виводить лише перші 5 рядків файлу. |
| `tail file.txt` | Виводить останні 10 рядків файлу. |
| `tail -f /var/log/syslog` | Виводить останні рядки файлу і продовжує відстежувати його зміни в реальному часі (зручно для читання логів). |
| `cut -d':' -f1 /etc/passwd` | Витягує певні стовпці з файлу. Тут: використовує двокрапку як роздільник (`-d':'`) і виводить лише перше поле (`-f1`). |
| `sort file.txt` | Сортує рядки у текстовому файлі в алфавітному порядку. |
| `sort -r file.txt` | Сортує рядки у зворотному (реверсному) алфавітному порядку. |
| `wc file.txt` | Виводить три значення для файлу: кількість рядків, кількість слів та кількість байтів. |
| `wc -l file.txt` | Виводить лише кількість рядків у файлі. |
| `grep "root" /etc/passwd` | Шукає та виводить всі рядки у файлі, які містять заданий шаблон (у цьому випадку слово "root"). |
| `grep -i "error" log.txt` | Шукає слово "error", ігноруючи регістр літер (знайде Error, ERROR, error). |
| `tr 'a-z' 'A-Z' < file.txt` | Замінює символи. У даному прикладі перетворює всі малі літери тексту з файлу на великі. |

## 3. Робота з командою `tar`


* **Створити файл з розширенням .tar:**
  `tar -cvf archive.tar /шлях/до/файлу_або_каталогу`
* **Створити файл з розширенням .tar, що складається з декількох файлів і каталогів одночасно:**
  `tar -cvf archive.tar file1.txt file2.txt /шлях/до/каталогу/`
* **Перегляд вмісту файлу .tar:**
  `tar -tvf archive.tar`
* **Витягти вміст файлу .tar:**
  `tar -xvf archive.tar`
* **Створити архівний файл .tar, стиснений за допомогою bzip2:**
  `tar -cjvf archive.tar.bz2 /шлях/до/файлу_або_каталогу`
* **Витягти вміст файлу .tar bzip2:**
  `tar -xjvf archive.tar.bz2`
* **Створити архівний файл .tar, стиснений за допомогою gzip:**
  `tar -czvf archive.tar.gz /шлях/до/файлу_або_каталогу`
* **Витягти вміст файлу .tar gzip:**
  `tar -xzvf archive.tar.gz`

## 4. Перенаправлення потоків виведення в bash

| Команда | Що виконує команда? |
| :--- | :--- |
| `cmd 1> file` | Перенаправляє стандартний потік виведення (`stdout`) у файл. Якщо файл існує, його вміст перезаписується. |
| `cmd > file` | Аналогічно до попередньої. За замовчуванням `>` діє як `1>`. Перезаписує файл. |
| `cmd 2> file` | Перенаправляє стандартний потік помилок (`stderr`) у файл. Перезаписує файл. |
| `cmd >> file` | Перенаправляє `stdout` у файл, але **додає** дані в кінець, не перезаписуючи попередній вміст. |
| `cmd &> file` | Перенаправляє одночасно і `stdout`, і `stderr` в один і той самий файл (перезаписуючи його). |
| `cmd > file 2>&1` | Спочатку направляє `stdout` у файл, а потім направляє `stderr` туди ж, куди направлений `stdout` (тобто у файл). |
| `cmd >> file 2>&1` | Додає `stdout` в кінець файлу, і туди ж перенаправляє помилки (`stderr`). |
| `cmd 2>&1 > /dev/null` | Направляє `stderr` туди, куди спочатку дивився `stdout` (на екран), а сам `stdout` відправляє у `/dev/null` (ігнорує). |
| `cmd 2> /dev/null` | Перенаправляє помилки (`stderr`) у `/dev/null` (приховує їх), при цьому звичайний вивід (`stdout`) відображається. |
| `cmd1 | cmd2` | **Канал (pipe):** передає стандартний вивід (`stdout`) команди `cmd1` на стандартне введення (`stdin`) команди `cmd2`. |
| `cmd1 2>&1 | cmd2` | Передає і звичайний вивід (`stdout`), і помилки (`stderr`) команди `cmd1` на вхід (`stdin`) команди `cmd2`. |

## 5. Приклади використання команд та перенаправлення потоків

| Команда (контейнер команд) | Що виконує команда? | Який потік перенаправлення? |
| :--- | :--- | :--- |
| `echo "It is a new story." > story` | Записує рядок "It is a new story." у файл `story` (перезаписуючи його). | Стандартне виведення (`stdout`) |
| `date > date.txt` | Виконує команду `date` (поточна дата і час) і зберігає результат у файл `date.txt`. | Стандартне виведення (`stdout`) |
| `cat file1 file2 file3 > bigfile` | Зчитує вміст трьох файлів, об'єднує їх та записує у новий файл `bigfile`. | Стандартне виведення (`stdout`) |
| `ls -l >> directory` | Виводить детальний список файлів і **додає** його в кінець файлу `directory`. | Стандартне виведення (`stdout`, додавання) |
| `sort < file1 unsorted > file2 sorted` | Бере невідсортовані дані з `file1 unsorted`, сортує їх і записує результат у `file2 sorted`. | Введення (`stdin`) та виведення (`stdout`) |
| `find -name '*.txt' > file.txt 2> /dev/null` | Шукає всі `.txt` файли. Результати пише у `file.txt`, а повідомлення про помилки доступу приховує. | Виведення (`stdout`) та помилки (`stderr`) |
| `cat file1 unsorted | sort > file2 sorted` | Читає файл, передає його вміст через канал команді `sort`, яка сортує дані і записує їх у новий файл. | Канал (`pipe`) та виведення (`stdout`) |
| `cat myfile | grep student | wc -l` | Читає файл, передає дані для пошуку рядків зі словом "student", а потім рахує їх кількість. | Канали (`pipe`) |

# Контрольні питання

---

## 1. Порівняльна характеристика процесів стискання та архівування

Хоча ці терміни часто використовують як синоніми, технічно це різні операції:

* **Архівування** — це процес об’єднання декількох файлів або каталогів в один єдиний файл (архів). Основна мета — спростити транспортування та зберігання даних, зберігаючи структуру папок і права доступу. Саме по собі архівування (наприклад, у форматі `.tar`) не зменшує розмір файлів.
* **Стискання (компресія)** — це процес зменшення фізичного обсягу даних за допомогою математичних алгоритмів, що усувають надлишковість.

| Характеристика | Архівування | Стискання |
| :--- | :--- | :--- |
| **Мета** | Об'єднання та впорядкування | Економія дискового простору |
| **Результат** | Один файл замість багатьох | Файл меншого розміру |
| **Метадані** | Зберігає права доступу, власників | Працює з потоком байтів |

---

## 2. Програми для Linux (окрім стандартних tar, gzip, bzip2)

Окрім базових утиліт, у Linux широко використовуються:

1.  **7-Zip (p7zip):** Кросплатформний інструмент з дуже високим ступенем стиснення. Використовує формат `.7z`.
2.  **Zstandard (zstd):** Розроблений компанією Facebook. Забезпечує надшвидке стиснення та розпакування, ідеально підходить для реального часу.
3.  **Lzop:** Базується на алгоритмі LZO. Це один із найшвидших архіваторів, де пріоритетом є швидкість, а не рівень стиснення.
4.  **Rar / Unrar:** Пропрієтарний формат, який часто використовується для сумісності з Windows-користувачами.

---

## 3. Порівняння алгоритмів стискання в Linux

В екосистемі Linux зазвичай виділяють три основні алгоритми:

* **Deflate (gzip):** Стандарт де-факто. Хороший баланс між швидкістю та рівнем стиснення.
* **BWT (bzip2):** Використовує перетворення Барроуза-Вілера. Стискає краще за gzip, але працює помітно повільніше.
* **LZMA/LZMA2 (xz):** Забезпечує найвищий рівень стиснення, але потребує значних ресурсів процесора та оперативної пам'яті.

> **Висновок:**
> * **Найшвидший:** `lzop` або `zstd`.
> * **Найефективніший (максимальне стиснення):** `xz` (LZMA2).

## 4. Програмні засоби для мобільних телефонів

На смартфонах (Android/iOS) функції архівування зазвичай інтегровані у файлові менеджери:

* **ZArchiver (Android):** Один із найпотужніших інструментів, що підтримує багатопотокове стиснення та величезну кількість форматів.
* **RAR (від RARLAB):** Офіційний порт WinRAR для Android.
* **Files by Google / iOS Files:** Вбудовані додатки, які мають базову підтримку створення та розпакування `.zip` архівів.

---

## 5. Програмні засоби для ОС Windows

У Windows домінують рішення з графічним інтерфейсом:

* **WinRAR:** Найпопулярніший платний архіватор. Підтримує формат `.rar`, який має високу стійкість до пошкоджень (завдяки блокам відновлення).
* **7-Zip:** Найкраща безкоштовна альтернатива з відкритим кодом. Його формат `.7z` часто перевершує за ефективністю комерційні аналоги.
* **Вбудований провідник:** Windows має рідну підтримку "стиснутих ZIP-папок", проте вона обмежена лише форматом `.zip`.

## 6. Використання для резервування та системного адміністрування

### Резервування даних (Backup)
Стиснення та архівування є фундаментом бекапу, оскільки:
* **Цілісність:** Архів об'єднує тисячі дрібних файлів у один, що легше копіювати на зовнішні носії.
* **Економія місця:** Дозволяє зберігати більше версій копій на тому ж самому обсязі диска.

### Інші задачі адміністрування:
* **Ротація логів:** Автоматичне стиснення старих журналів подій (`logrotate`) для запобігання переповненню диска.
* **Передача даних:** Зменшення мережевого трафіку при використанні утиліт на кшталт `rsync` з прапором стиснення.
* **Дистрибуція ПЗ:** Пакування програмного коду або бінарних пакетів у "тарболи" (`.tar.gz`, `.tar.xz`) для розповсюдження.

---

## 7. Призначення директорії файлу /dev/null

**/dev/null** — це спеціальний віртуальний пристрій у Linux, який часто називають "чорною дірою".

* **Запис у /dev/null:** Будь-які дані, спрямовані туди, миттєво зникають. Використовується для придушення непотрібного виводу команд (наприклад: `command > /dev/null`).
* **Читання з /dev/null:** При спробі читання він одразу повертає символ кінця файлу (EOF), тобто "нічого".
