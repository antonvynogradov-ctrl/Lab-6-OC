## 📑 Розширений глосарій технічних термінів

Для професійного адміністрування та ефективної роботи з даними в Linux необхідно розрізняти такі фундаментальні поняття:

* **Архівування (Archiving)** — операція об'єднання множини окремих файлів, каталогів та їх метаданих (права доступу, власники, часові мітки) у єдиний контейнер-файл. Це спрощує транспортування даних, але саме по собі не зменшує їх фізичний обсяг.
* **Стиснення (Compression)** — застосування складних математичних алгоритмів (наприклад, LZ77, Huffman або BWT) для усунення надлишковості в коді файлу з метою мінімізації займаного дискового простору.
* **Командний інтерпретатор (Shell)** — програмне середовище (наприклад, Bash), яке виступає посередником між користувачем та ядром ОС, обробляючи текстові інструкції та керуючи процесами.
* **Стандартні потоки (I/O Streams)** — системні канали передачі даних: `stdin` (0) — вхідний потік, `stdout` (1) — стандартний вихід, `stderr` (2) — потік виводу системних помилок.
* **Перенаправлення (Redirection)** — механізм зміни цільового призначення потоків, що дозволяє, наприклад, записувати результат роботи команди у файл замість виводу на екран.
* **Конвеєризація (Piping)** — унікальна особливість UNIX-подібних систем, що дозволяє передавати вихідні дані однієї утиліти безпосередньо на вхід іншої, створюючи потужні ланцюжки обробки інформації.

| Утиліта | Технічний опис та функціональні можливості |
| :--- | :--- |
| **`tar`** | Легендарний інструмент ("Tape Archiver"). Здатний створювати, переглядати та розпаковувати архіви. Підтримує інтеграцію з алгоритмами стиснення через спеціальні прапори. |
| **`gzip` / `gunzip`** | Класичний інструмент на базі алгоритму Deflate. Забезпечує високу швидкість роботи при гарному рівні стиснення. Формат файлів — `.gz`. |

## 📦 3. Глибока специфікація роботи з утилітою `tar`

Команда `tar` є "швейцарським ножем" для системного адміністратора. Ми відпрацювали такі ключові сценарії:

* **Створення простого архіву:** `tar -cvf archive.tar /path` (ключ `-c` — create).
* **Аудит без розпакування:** `tar -tvf archive.tar` (ключ `-t` — list).
* **Стиснення GZIP:** `tar -czvf archive.tar.gz /path` (ключ `-z` підключає gzip).
* **Стиснення BZIP2:** `tar -cjvf archive.tar.bz2 /path` (ключ `-j` підключає bzip2).
* **Розпакування:** `tar -xvf archive.tar` (ключ `-x` — extract).

## 🌊 4. Механіка перенаправлення та конвеєризації в Bash

Взаємодія з потоками I/O дозволяє автоматизувати найскладніші завдання:

| Оператор | Механізм дії |
| :--- | :--- |
| **`>`** | Повний перезапис файлу результатом роботи команди. |
| **`>>`** | Дозапис нових даних у кінець файлу без видалення існуючих. |
| **`2>`** | Перенаправлення лише повідомлень про помилки (stderr). |
| **`&>`** | Об'єднання звичайного виводу та помилок в один файл. |
| **`/dev/null`** | Направлення даних у "віртуальну чорну діру" для їх повного ігнорування. |
| **`|` (Pipe)** | З'єднання виходу однієї програми з входом іншої. Основа філософії UNIX. |

## 1. Порівняльна характеристика процесів стискання та архівування

Хоча ці терміни часто використовують як синоніми, технічно це різні операції:

* **Архівування** — це процес об’єднання декількох файлів або каталогів в один єдиний файл (архів). Основна мета — спростити транспортування та зберігання даних, зберігаючи структуру папок і права доступу. Саме по собі архівування (наприклад, у форматі `.tar`) не зменшує розмір файлів.
* **Стискання (компресія)** — це процес зменшення фізичного обсягу даних за допомогою математичних алгоритмів, що усувають надлишковість.

| Характеристика | Архівування | Стискання |
| :--- | :--- | :--- |
| **Мета** | Об'єднання та впорядкування | Економія дискового простору |
| **Результат** | Один файл замість багатьох | Файл меншого розміру |
| **Метадані** | Зберігає права доступу, власників | Працює з потоком байтів |

---

## 2. Програми для Linux (окрім стандартних tar, gzip, bzip2)

Окрім базових утиліт, у Linux широко використовуються:

1.  **7-Zip (p7zip):** Кросплатформний інструмент з дуже високим ступенем стиснення. Використовує формат `.7z`.
2.  **Zstandard (zstd):** Розроблений компанією Facebook. Забезпечує надшвидке стиснення та розпакування, ідеально підходить для реального часу.
3.  **Lzop:** Базується на алгоритмі LZO. Це один із найшвидших архіваторів, де пріоритетом є швидкість, а не рівень стиснення.
4.  **Rar / Unrar:** Пропрієтарний формат, який часто використовується для сумісності з Windows-користувачами.

---

## 3. Порівняння алгоритмів стискання в Linux

В екосистемі Linux зазвичай виділяють три основні алгоритми:

* **Deflate (gzip):** Стандарт де-факто. Хороший баланс між швидкістю та рівнем стиснення.
* **BWT (bzip2):** Використовує перетворення Барроуза-Вілера. Стискає краще за gzip, але працює помітно повільніше.
* **LZMA/LZMA2 (xz):** Забезпечує найвищий рівень стиснення, але потребує значних ресурсів процесора та оперативної пам'яті.

> **Висновок:**
> * **Найшвидший:** `lzop` або `zstd`.
> * **Найефективніший (максимальне стиснення):** `xz` (LZMA2).

## 5. Програмні засоби для ОС Windows

У Windows домінують рішення з графічним інтерфейсом:

* **WinRAR:** Найпопулярніший платний архіватор. Підтримує формат `.rar`, який має високу стійкість до пошкоджень (завдяки блокам відновлення).
* **7-Zip:** Найкраща безкоштовна альтернатива з відкритим кодом. Його формат `.7z` часто перевершує за ефективністю комерційні аналоги.
* **Вбудований провідник:** Windows має рідну підтримку "стиснутих ZIP-папок", проте вона обмежена лише форматом `.zip`.

## 6. Використання для резервування та системного адміністрування

### Резервування даних (Backup)
Стиснення та архівування є фундаментом бекапу, оскільки:
* **Цілісність:** Архів об'єднує тисячі дрібних файлів у один, що легше копіювати на зовнішні носії.
* **Економія місця:** Дозволяє зберігати більше версій копій на тому ж самому обсязі диска.

### Інші задачі адміністрування:
* **Ротація логів:** Автоматичне стиснення старих журналів подій (`logrotate`) для запобігання переповненню диска.
* **Передача даних:** Зменшення мережевого трафіку при використанні утиліт на кшталт `rsync` з прапором стиснення.
* **Дистрибуція ПЗ:** Пакування програмного коду або бінарних пакетів у "тарболи" (`.tar.gz`, `.tar.xz`) для розповсюдження.

## 7. Призначення директорії файлу /dev/null

**/dev/null** — це спеціальний віртуальний пристрій у Linux, який часто називають "чорною дірою".

* **Запис у /dev/null:** Будь-які дані, спрямовані туди, миттєво зникають. Використовується для придушення непотрібного виводу команд (наприклад: `command > /dev/null`).
* **Читання з /dev/null:** При спробі читання він одразу повертає символ кінця файлу (EOF), тобто "нічого".
