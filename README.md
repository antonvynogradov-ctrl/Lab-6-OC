## Розширений глосарій технічних термінів

Для професійного адміністрування та ефективної роботи з даними в Linux необхідно розрізняти такі фундаментальні поняття:

* **Архівування (Archiving)** — операція об'єднання множини окремих файлів, каталогів та їх метаданих (права доступу, власники, часові мітки) у єдиний контейнер-файл. Це спрощує транспортування даних, але саме по собі не зменшує їх фізичний обсяг.
* **Стиснення (Compression)** — застосування складних математичних алгоритмів (наприклад, LZ77, Huffman або BWT) для усунення надлишковості в коді файлу з метою мінімізації займаного дискового простору.
* **Командний інтерпретатор (Shell)** — програмне середовище (наприклад, Bash), яке виступає посередником між користувачем та ядром ОС, обробляючи текстові інструкції та керуючи процесами.
* **Стандартні потоки (I/O Streams)** — системні канали передачі даних: `stdin` (0) — вхідний потік, `stdout` (1) — стандартний вихід, `stderr` (2) — потік виводу системних помилок.
* **Перенаправлення (Redirection)** — механізм зміни цільового призначення потоків, що дозволяє, наприклад, записувати результат роботи команди у файл замість виводу на екран.
* **Конвеєризація (Piping)** — унікальна особливість UNIX-подібних систем, що дозволяє передавати вихідні дані однієї утиліти безпосередньо на вхід іншої, створюючи потужні ланцюжки обробки інформації.

## 2. Комплексний аналіз інструментарію Linux

У ході лабораторної роботи ми дослідили та протестували наступний набір утиліт:

### А. Керування архівами та компресією

| Утиліта | Технічний опис та функціональні можливості |
| :--- | :--- |
| **`tar`** | Легендарний інструмент ("Tape Archiver"). Здатний створювати, переглядати та розпаковувати архіви. Підтримує інтеграцію з алгоритмами стиснення через спеціальні прапори. |
| **`gzip` / `gunzip`** | Класичний інструмент на базі алгоритму Deflate. Забезпечує високу швидкість роботи при гарному рівні стиснення. Формат файлів — `.gz`. |
| **`bzip2` / `bunzip2`** | Використовує перетворення Барроуза-Вілера. Працює повільніше за gzip, але демонструє вищу щільність пакування, особливо для текстів. Формат — `.bz2`. |
| **`xz` / `unxz`** | Сучасний лідер на основі алгоритму LZMA2. Забезпечує найвищий рівень стиснення, проте вимагає значних ресурсів процесора та пам'яті. Формат — `.xz`. |
| **`zip` / `unzip`** | Забезпечує кросплатформну сумісність. Вміє архівувати та стискати дані одночасно, що зручно для обміну файлами з користувачами Windows. |

### Б. Текстові процесори та фільтри

| Команда | Призначення в професійному робочому процесі |
| :--- | :--- |
| **`cat`** | Конкатенація та швидкий вивід вмісту файлів на екран. |
| **`less`** | Розумний переглядач великих файлів. Дозволяє навігацію в обох напрямках без повного завантаження файлу в оперативну пам'ять. |
| **`grep`** | Потужний інструмент пошуку за шаблонами. Незамінний для аналізу гігантських лог-файлів та пошуку помилок. |
| **`head` / `tail`** | Перегляд початку або кінця файлу. Команда `tail -f` є критично важливою для моніторингу системних журналів у реальному часі. |
| **`sort` / `uniq`** | Впорядкування даних в алфавітному чи числовому порядку та видалення ідентичних рядків. |
| **`cut` / `tr`** | Інструменти для вирізання окремих стовпців тексту або заміни одних символів іншими (наприклад, зміна регістру). |
| **`wc`** | Утиліта підрахунку статистики: кількості рядків (`-l`), слів (`-w`) та байтів (`-c`). |

## 3. Глибока специфікація роботи з утилітою `tar`

Команда `tar` є "швейцарським ножем" для системного адміністратора. Ми відпрацювали такі ключові сценарії:

* **Створення простого архіву:** `tar -cvf archive.tar /path` (ключ `-c` — create).
* **Аудит без розпакування:** `tar -tvf archive.tar` (ключ `-t` — list).
* **Стиснення GZIP:** `tar -czvf archive.tar.gz /path` (ключ `-z` підключає gzip).
* **Стиснення BZIP2:** `tar -cjvf archive.tar.bz2 /path` (ключ `-j` підключає bzip2).
* **Розпакування:** `tar -xvf archive.tar` (ключ `-x` — extract).

## 4. Механіка перенаправлення та конвеєризації в Bash

Взаємодія з потоками I/O дозволяє автоматизувати найскладніші завдання:

| Оператор | Механізм дії |
| :--- | :--- |
| **`>`** | Повний перезапис файлу результатом роботи команди. |
| **`>>`** | Дозапис нових даних у кінець файлу без видалення існуючих. |
| **`2>`** | Перенаправлення лише повідомлень про помилки (stderr). |
| **`&>`** | Об'єднання звичайного виводу та помилок в один файл. |
| **`/dev/null`** | Направлення даних у "віртуальну чорну діру" для їх повного ігнорування. |
| **`|` (Pipe)** | З'єднання виходу однієї програми з входом іншої. Основа філософії UNIX. |

 РОЗШИРЕНІ ВІДПОВІДІ НА КОНТРОЛЬНІ ЗАПИТАННЯ
1. Архівування vs Стискання: Архівування — це процес збору файлів у "контейнер" для зручності. Стискання — це зміна кодування даних для зменшення їх ваги. Формат .tar сам по собі не стискає дані, він лише склеює їх.

2. Додаткові утиліти: Окрім стандартних, існують 7-Zip (висока компресія), Zstandard (неймовірна швидкість від Facebook) та Lzop (мінімальне навантаження на процесор).

3. Ефективність алгоритмів: Найшвидшим є lzop, найефективнішим за розміром — xz (LZMA2), а золотою серединою залишається gzip.

4. Мобільні ОС: В Android та iOS популярними є ZArchiver та RAR. Самі додатки .apk та .ipa за своєю суттю є модифікованими ZIP-архівами.

5. ОС Windows: Тут домінують WinRAR та 7-Zip. Формат .rar цінується за можливість відновлення пошкоджених даних, а .7z — за відкритий код та безкоштовність.

6. Значення /dev/null: Це системна "чорна діра". Направляючи туди вивід (> /dev/null), ми змушуємо систему ігнорувати його. Це корисно для прихованого виконання фонових скриптів.
